# mashtree

**Tags:** <a href="/developers/tags/phylogeny" class="badge badge--primary badge--tag">phylogeny</a> <a href="/developers/tags/tree" class="badge badge--primary badge--tag">tree</a> <a href="/developers/tags/mash" class="badge badge--primary badge--tag">mash</a> <a href="/developers/tags/minhash" class="badge badge--primary badge--tag">minhash</a> <a href="/developers/tags/alignment-free" class="badge badge--primary badge--tag">alignment-free</a> <a href="/developers/tags/distance" class="badge badge--primary badge--tag">distance</a> <a href="/developers/tags/clustering" class="badge badge--primary badge--tag">clustering</a> <a href="/developers/tags/neighbor-joining" class="badge badge--primary badge--tag">neighbor-joining</a> <a href="/developers/tags/run-scope" class="badge badge--primary badge--tag">run-scope</a> 

Construção rápida de árvores filogenômicas sem alinhamento.

Utiliza o [Mashtree](https://github.com/lskatz/mashtree) para criar uma árvore filogenética
a partir de sequências genômicas (FASTA, FASTQ ou GenBank) usando distâncias MinHash. O módulo
calcula distâncias pareadas entre todas as entradas e usa o algoritmo Neighbor-Joining para
agrupar os genomas, criando efetivamente uma árvore "baseada em distâncias" sem alinhamento completo.

## Entradas

```
record (
    meta: Record,
    fna: Set<Path>
)
```

| Campo | Tipo | Descrição |
|-------|------|-----------|
| `meta` | `Record` | Registro Groovy contendo informações da amostra |
| `fna` | `Set<Path>` | Contigs montados no formato FASTA |

## Saídas

```
record (
    meta: Record,
    nwk: Path,
    tsv: Path,
    sketches: Set<Path?>,
    results: Set<Path>,
    logs: Set<Path?>,
    nf_logs: Set<Path>,
    versions: Set<Path>
)
```

| Campo | Tipo | Descrição |
|-------|------|-----------|
| `meta` | `Record` | Registro com informações da amostra |
| `nwk` | `Path` | A árvore filogenética final no formato Newick (*.dnd) |
| `tsv` | `Path` | A matriz de distâncias pareadas usada para construir a árvore (*.tsv) |
| `sketches` | `Set<Path?>` | Diretório contendo os sketches Mash individuais |
| `results` | `Set<Path>` | Todos os arquivos de saída a serem publicados |
| `logs` | `Set<Path?>` | Arquivos de log opcionais específicos do programa |
| `nf_logs` | `Set<Path>` | Arquivos de log específicos do Nextflow (ex.: .command.&#123;begin|err|log|out|run|sh|trace&#125;) |
| `versions` | `Set<Path>` | Arquivo no formato YAML com as versões dos programas |

## Parâmetros

### Parâmetros do Mashtree

| Parâmetro | Tipo | Padrão | Descrição |
|-----------|------|--------|-----------|
| `--mashtree_trunclength` | inteiro | `250` | Quantidade de caracteres a manter em um nome de arquivo |
| `--mashtree_sortorder` | string | `ABC` | Para o Neighbor-Joining, a ordem de classificação pode fazer diferença. (opções: `ABC`, `random`, `input-order`) |
| `--mashtree_genomesize` | inteiro | `5000000` | Tamanho do genoma das amostras de entrada |
| `--mashtree_mindepth` | inteiro | `5` | Se mindepth for zero, o valor será escolhido de forma inteligente, porém mais lenta, para descartar kmers de baixa abundância. |
| `--mashtree_kmerlength` | inteiro | `21` | Os hashes serão baseados em strings com esse número de nucleotídeos |
| `--mashtree_sketchsize` | inteiro | `10000` | Cada sketch terá no máximo esse número de min-hashes não redundantes |
| `--mashtree_save_sketches` | booleano | `false` | Salvar os sketches criados durante o processo |
