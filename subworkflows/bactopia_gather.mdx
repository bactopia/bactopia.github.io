# bactopia_gather

<span class="badge badge--success">stable</span> <span class="badge badge--info">sample scope</span>

Search, validate, gather, and standardize input samples.

This subworkflow processes raw input samples through validation, standardization, and metadata
collection. It handles various input types including local FASTQ files, SRA/ENA accessions,
NCBI assembly accessions, and assemblies. The workflow can merge multiple sequencing runs,
download remote data, and simulate reads from assemblies using [ART](https://www.niehs.nih.gov/research/resources/software/biostatistics/art).

Uses explicit positional record fields for reads:
- Input: record(meta, r1_files, r2_files, se_files, lr_files) with Set&lt;Path&gt; slots (pre-merge)
- Output: record(meta, r1, r2, se, lr) with Path? slots (post-merge, consolidated)

## Take

```
samples: Channel<Record>
```

| Field | Description |
|-------|-------------|
| `meta` | Groovy Record containing sample information |
| `r1_files` | Illumina R1 read files (Set, elements may be null) |
| `r2_files` | Illumina R2 read files (Set, elements may be null) |
| `se_files` | Single-end read files (Set, elements may be null) |
| `lr_files` | Long read files (ONT/PacBio) or assembly for simulation (Set, elements may be null) |

## Emit

### Published

The `sample_outputs` and `run_outputs` emissions are aggregates of output files that will be published in the entry workflow.

#### `sample_outputs`

| Output | Description |
|--------|-------------|
| `tsv` | A tab-delimited metadata file describing the valid samples |

#### `run_outputs`

| Output | Description |
|--------|-------------|
| `csv` | Aggregated metadata from all samples |

### Downstream Inputs

The following emissions are meant to be used as inputs to downstream subworkflows.

#### `reads`

| Output | Description |
|--------|-------------|
| `r1` | Illumina R1 reads (paired-end forward) |
| `r2` | Illumina R2 reads (paired-end reverse) |
| `se` | Single-end Illumina reads |
| `lr` | Long reads (ONT/PacBio) |
| `fna` | Assembly file for assembly-based samples |

## Module Composition

This subworkflow calls the following modules:

- [bactopia_gather](/modules/bactopia_gather) - Search, validate, gather, or simulate input samples.
- [csvtk_concat](/modules/csvtk_concat) - Concatenate multiple CSV or TSV files into a single table.

## Used By

This subworkflow is used by the following workflows:

- [bactopia](/workflows/named-workflows/bactopia) - Comprehensive bacterial analysis pipeline for complete genomic characterization.
- [cleanyerreads](/workflows/named-workflows/cleanyerreads) - Quality control and optional host read removal from raw sequencing reads.
- [staphopia](/workflows/named-workflows/staphopia) - Comprehensive analysis pipeline for Staphylococcus aureus isolates.
- [teton](/workflows/named-workflows/teton) - Taxonomic classification and abundance profiling of metagenomic reads.
