# shigeifinder

**Tags:** <a href="/developers/tags/shigella" class="badge badge--primary badge--tag">shigella</a> <a href="/developers/tags/eiec" class="badge badge--primary badge--tag">eiec</a> <a href="/developers/tags/serotype" class="badge badge--primary badge--tag">serotype</a> <a href="/developers/tags/typing" class="badge badge--primary badge--tag">typing</a> <a href="/developers/tags/cluster-analysis" class="badge badge--primary badge--tag">cluster-analysis</a> <a href="/developers/tags/sample-scope" class="badge badge--primary badge--tag">sample-scope</a> 

Predict serotypes of Shigella and EIEC from assemblies.

This subworkflow uses [ShigEiFinder](https://github.com/LanLab/ShigEiFinder) to predict
serotypes of *Shigella* and Enteroinvasive *E. coli* (EIEC) from assembled genomes.
It uses a cluster-informed approach to identify specific serotype markers and classify
isolates based on their antigenic profiles.

## Take

```
assembly: Channel<Record>
```

| Field | Description |
|-------|-------------|
| `meta` | Groovy Record containing sample information |
| `assembly` | Assembled contigs in FASTA format |

## Emit

### Published

The `sample_outputs` and `run_outputs` emissions are aggregates of output files that will be published in the entry workflow.

#### `sample_outputs`

| Output | Description |
|--------|-------------|
| `tsv` | ShigEiFinder results in TSV format |

#### `run_outputs`

| Output | Description |
|--------|-------------|
| `csv` | Aggregated results in CSV format |

## Module Composition

This subworkflow calls the following modules:
